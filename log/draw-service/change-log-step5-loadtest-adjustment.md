# Step 5 부하 테스트 지갑 데이터 분포 보정

## 2026-05-10 20:25:00 +09:00 - 부하 테스트 지갑 데이터 분포 보정

### 1. 발생 상황
- 자동화 스크립트를 다시 실행했을 때 재고가 거의 차감되지 않고 결과가 비정상적으로 보이는 상황이 발생했다.
- k6 요청은 `userId`를 1번부터 순서대로 생성하는데, 기존 지갑 시딩은 앞쪽 1000명을 모두 잔액 부족 사용자로 만들고 있었다.

### 2. 원인
- 첫 1000명 단계에서는 `userId=1~1000`이 전부 5000원 지갑이어서 가격 10000원 결제에 실패했다.
- draw-service는 선점 후 결제 실패 시 보상 트랜잭션으로 재고를 다시 +1 하므로, 재고가 거의 줄지 않는 결과가 정상적으로 발생할 수 있었다.

### 3. 조치
- 자동화 스크립트의 기본 지갑 수를 5000명으로 늘렸다.
- 잔액 부족 사용자와 잔액 충분 사용자가 앞쪽 요청 구간에도 섞이도록 지갑 시딩을 홀짝 분산 방식으로 변경했다.
- 기본 요청 단계 1000, 2000, 3000, 5000명 모두에서 결제 성공/실패/품절 흐름을 관측할 수 있게 했다.
