- `payment` 도메인에 `POST /api/v1/payments` 차감 API와 `requestId` 기반 중복 요청 방지 기록을 추가했다.
- 공통 응답 포맷, 비즈니스 예외, 전역 예외 처리, Trace ID 필터, JPA Auditing 기반 `BaseEntity`를 추가했다.
- 잔액 차감 성공, 잔액 부족 롤백, 중복 결제 요청 방지, 엔티티 차감 규칙을 검증하는 테스트를 추가했다.
- 현재 단계 지침에 맞춰 `@Lock`, `@Version`, Redis 원자 연산 등 동시성 제어 코드는 추가하지 않았다.

## 2026-05-09 17:05:00 KST - 루트 Gradle core plugin 선언 오류 수정

### 1. 발생 상황
- `.\gradlew :payment-service:build -x test` 실행 시 루트 `build.gradle` 2번째 줄에서 `Plugin 'java' is a core Gradle plugin` 오류가 발생했다.

### 2. 원인
- Gradle core plugin인 `java`는 이미 Gradle 자체 classpath에 포함되어 있어 루트 `plugins` 블록에서 `apply false`로 선언할 필요가 없다.
- Gradle 8.14 계열에서는 이 선언을 no-op으로 처리하지 않고 빌드 실패로 보고한다.

### 3. 조치
- 루트 `build.gradle`의 `id 'java' apply false` 선언을 제거했다.
- Spring Boot 및 dependency-management 플러그인 버전 관리는 루트에서 유지하고, `java` 플러그인은 각 하위 모듈에서 직접 적용하도록 유지했다.

- 존재하지 않는 경로는 `INTERNAL_SERVER_ERROR`가 아니라 `NOT_FOUND` 404 응답으로 내려가도록 예외 처리기를 보강했다.
